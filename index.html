👨‍💻 ABOUT THE PROJECT
IoT-Driven Smart Meal Quality Tracking
🥗
IoT-Driven Smart Meal Quality Tracking And Evaluation System

Our project monitors meal quality using sensors and image-based analysis. We use the MQ-135 Gas Sensor for spoilage smell, MQ-3 Gas Sensor for smoke/combustion, TCS3200 Color Sensor for visual analysis, and DHT11 Sensor for temperature/humidity. The AI Camera module uses a trained MobileNetV2 CNN to detect fungus and classify food as FRESH or NOT FRESH.

⚙️ SYSTEM COMPONENTS
☁️
MQ-135 Aroma Sensor
AIR QUALITY / SPOILAGE DETECTION

0–25% → FRESH, 26–55% → WARNING, 56–100% → SPOILED.

🔥
MQ-3 Smoke/Gas Sensor
SMOKE / OVERCOOKING DETECTION

0–20% → NORMAL, 21–50% → WARNING, 51–100% → OVERCOOKED.

🎨
TCS3200 Color Sensor
VISUAL FOOD CONDITION ANALYSIS

RGB values identify visual condition. Green shift = mold, dark brown = overcooked.

🌡️
DHT11 Temp & Humidity
FOOD PRESERVATION ANALYSIS

Rice ~72°C, Puff ~65°C, Chicken ~85°C. Humidity: <65% NORMAL.

🧠
AI Image Analysis (CNN)
FUNGUS / MOLD DETECTION

MobileNetV2 — 93.8% accuracy, 5 epochs, 2400 images. Supports USB webcam + built-in cameras.

💡 Project Goal: Build a reliable, automated system that helps mess administrators and students monitor food safety in real time — reducing health risks caused by spoiled or sub-standard food.
